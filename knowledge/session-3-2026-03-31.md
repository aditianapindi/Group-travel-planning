## Decisions made
- **Holiday calendar**: Designed but NOT built. Documented in knowledge/decision-holiday-calendar.md. Right idea, wrong moment — core flow needed verification first. Describe in PRD.
- **Participant motivation**: Built. Documented in knowledge/decision-participant-motivation.md. Behavioral levers: named accountability > deadline consequence > tiny ask > instant feedback.
- **Trip probability meter**: Rejected. Can't calculate honestly, feels like fake gamification.
- **Destination price estimates**: Deferred. Would need Gemini API call at trip creation + schema change. Describe in PRD.
- **Dark mode**: Skipped for deadline. Not expected for prototype submission.
- **Polling vs Realtime**: Switched from polling to Supabase Realtime. Polling was the wrong tool — wasteful at scale, Supabase Realtime is built into the SDK.

## Lessons learned
- Verify rendered output for each persona before handoff (caught "Aditican" space bug, wrong tone for organizer, duplicate names)
- "Do less. Verify more" — almost built holiday calendar (3-4 hrs) when core flow wasn't stress-tested
- Push back on own recommendations, not just user's requests
